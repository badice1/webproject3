四、软件的运行与结果
4.1 系统运行测试过程
4.1.1 测试环境
操作系统：Windows 10/11、macOS Ventura、Linux Ubuntu 22.04；
浏览器：Chrome 120+、Firefox 119+、Safari 17+、Edge 120+；
硬件配置：CPU i5-10400F、内存 16GB、硬盘 1TB（测试机配置）；
网络环境：100Mbps 有线网络、5G 无线网络；
测试工具：JMeter（并发测试）、Chrome 开发者工具（性能测试）、Postman（接口测试）、Supabase 控制台（数据库测试）。
4.1.2 测试方法
功能测试：手工测试核心功能点（注册、登录、会员申请、审核流程、消息发送等），验证是否符合需求；利用 Vite 开发服务器实时预览修改效果，加速功能测试迭代；
性能测试：使用浏览器 Network 面板监控页面加载时间与 Supabase 接口响应时间，通过 JMeter 模拟 1000 用户并发访问，重点测试会员列表加载、申请提交等高频操作；
兼容性测试：在不同操作系统、浏览器、分辨率下测试系统运行情况，验证 Tailwind CSS 响应式设计的有效性，确保界面展示与功能可用性；
安全测试：测试权限控制有效性（非授权访问、角色越权），验证 ProtectedRoute 组件的保护机制；检查 Supabase RLS 策略是否有效，确保数据访问安全。
4.2 功能展示
4.2.1 登录 / 注册页面
登录界面：简洁的邮箱 / 密码输入表单，包含 "注册"、"忘记密码" 链接；
注册界面：邮箱、密码、确认密码输入表单，注册成功后自动跳转登录页；
忘记密码：输入注册邮箱，发送重置链接到邮箱；
重置密码：通过邮箱链接进入，设置新密码；
功能：输入合法信息可成功登录 / 注册，输入错误信息提示明确错误原因，如 "连接服务器失败"、"邮箱或密码错误" 等。
4.2.2 会员仪表盘
信息展示：显示会员个人基础信息、缴费状态、会员级别；
核心功能入口：会员申请、消息中心、事件公告、个人信息编辑。
4.2.3 管理员仪表盘
会员管理：展示所有会员列表，支持查看和编辑会员级别、角色等信息；
申请审核：查看会员入会申请，支持通过 / 拒绝操作，实时更新申请状态；
导出功能：点击 "导出 Excel" 按钮，使用 XLSX 库生成并下载会员列表文件；
通知功能：支持消息中心通知方式，可向会员发送通知；
事件管理：查看和管理活动信息，关联活动参与者。
4.2.4 核心页面
消息中心：展示会员接收和发送的所有消息，支持标记已读、删除消息，未读消息高亮提示；
事件公告板：按发布时间倒序展示活动，支持查看活动详情；
密码重置页面：验证邮箱后重置密码，流程简洁安全。
