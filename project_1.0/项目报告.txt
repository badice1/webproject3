广西自动化学会会员管理系统项目报告

一、需求分析与软件总体方案

1.1 需求分析以及设计要求

1.1.1 功能需求
用户认证：支持会员和管理员登录/注册，包含忘记密码功能
会员管理：会员信息管理、缴费状态管理、会员级别管理
申请审核：会员入会申请提交与审核流程
通知系统：消息中心、事件公告
数据导出：会员列表Excel导出功能
权限控制：基于角色的访问控制

1.1.2 性能需求
页面加载时间<2秒
支持1000+并发用户
数据存储安全可靠

1.1.3 设计要求
用户友好的界面设计
响应式布局，适配不同设备
清晰的操作流程
完善的错误处理和提示

1.2 总体的方案与技术路线

1.2.1 总体方案与说明
系统采用前后端分离架构：
前端：React 18 + TypeScript + Vite + Tailwind CSS
后端：Supabase（数据库、认证服务）
数据通信：RESTful API
部署方式：GitHub Pages静态部署

1.2.2 多种约束因素考虑
技术约束：使用已学技术栈，易于维护
成本约束：采用免费开源技术，降低成本
时间约束：快速开发，迭代优化
安全约束：数据加密存储，权限严格控制

1.2.3 详细技术路线
1. 需求分析：明确功能需求和非功能需求
2. 技术选型：确定前后端技术栈
3. 数据库设计：设计表结构和关系
4. 前端开发：组件化开发，实现页面功能
5. 后端配置：Supabase数据库和认证配置
6. 测试调试：功能测试、性能测试、兼容性测试
7. 部署上线：GitHub Pages部署

1.2.4 时间成本与人力成本估算
时间成本：4周（需求分析1周，设计1周，开发1周，测试部署1周）
人力成本：1人，约160工时

1.3 相关标准
Web标准：HTML5、CSS3、JavaScript ES6+
代码规范：ESLint、TypeScript类型检查
安全标准：HTTPS、数据加密、权限控制

二、软件设计

2.1 概要设计

2.1.1 数据结构
Profile：用户基本信息、角色、会员状态
Application：会员申请信息
AppMessage：消息通知
AppEvent：事件公告
EventParticipant：活动参与者

2.1.2 数据库设计
profiles表：存储用户核心信息，与auth.users关联
applications表：存储会员申请记录
messages表：存储系统消息
events表：存储事件公告
event_participants表：存储活动参与者信息

2.2 详细设计

2.2.1 程序框架
前端架构：组件化设计，基于React Hooks和Context API
路由设计：使用React Router实现路由管理，包含公开路由和受保护路由
状态管理：使用Context API管理全局状态
数据通信：通过Supabase客户端库与后端交互

2.2.2 核心功能模块设计

2.2.2.1 用户认证模块
登录流程：输入邮箱密码 → 验证凭证 → 判断角色 → 跳转到对应仪表盘
注册流程：输入注册信息 → 创建用户 → 初始化profile → 跳转到登录页

2.2.2.2 会员管理模块
信息查看：查询并展示会员信息
级别管理：管理员可修改会员级别
状态更新：更新会员缴费状态和会员状态

2.2.2.3 申请审核模块
申请提交：会员填写申请表格 → 提交申请 → 等待审核
申请审核：管理员查看申请 → 审核（通过/拒绝）→ 发送通知

2.2.2.4 数据导出模块
导出流程：查询会员数据 → 生成Excel文件 → 下载文件

三、软件实现与调试

3.1 存储结构设计说明
系统数据存储依赖Supabase的PostgreSQL数据库，采用关系型存储结构：
核心用户数据：存储于profiles表，与auth.users表通过外键关联
申请数据：存储于applications表，通过user_id字段与profiles表关联
消息数据：存储于messages表，通过sender_id和receiver_id双向关联profiles表
事件数据：存储于events表，关联管理员ID
参与者数据：存储于event_participants表，关联events表和profiles表

4.3 软件使用说明

4.3.1 安装与部署
1. 克隆代码仓库
2. 安装依赖：npm install
3. 配置环境变量：复制.env.example为.env，填写Supabase配置
4. 启动开发服务器：npm run dev
5. 构建生产版本：npm run build

4.3.2 用户使用指南

4.3.2.1 会员使用指南
1. 注册：访问注册页面，填写邮箱和密码
2. 登录：使用注册邮箱和密码登录
3. 查看个人信息：登录后在会员仪表盘查看
4. 申请会员：点击"申请会员"按钮，填写申请表格
5. 查看消息：在消息中心查看系统通知
6. 查看公告：在事件公告板查看活动公告

4.3.2.2 管理员使用指南
1. 登录：使用管理员账号登录
2. 管理会员：在管理员仪表盘查看和编辑会员信息
3. 审核申请：查看会员申请，进行通过/拒绝操作
4. 导出数据：点击"导出Excel"按钮下载会员列表
5. 管理事件：创建和管理活动公告

4.3.3 常见问题解答
忘记密码：点击登录页面的"忘记密码"链接，按照提示重置密码
无法登录：检查邮箱和密码是否正确，网络连接是否正常
申请未审核：耐心等待管理员审核，可在消息中心查看审核结果
